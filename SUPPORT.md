# Support

Lantern Protocol repositories use GitHub Issues for actionable project work, not general support.

## Use GitHub Issues for

- Reproducible bugs.
- Bounded feature requests.
- Protocol proposals.
- Documentation gaps.
- Build, test, or automation failures.

## Do not use public issues for

- Security vulnerabilities.
- Private operational details.
- Sensitive incident data.
- Credentials, tokens, logs, or personal data.
- Non-actionable general discussion.

Security-sensitive matters should follow `SECURITY.md`.

## Before opening an issue

Please search existing issues and include enough context for maintainers to evaluate the request without guessing.
