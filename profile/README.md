# Lantern Protocol

Lantern Protocol is an applied research and engineering organization focused on trustworthy signal infrastructure, adversarial-resilience, evidence provenance, and civic/emergency-condition data systems.

## Current repositories

- **signalforge** — early implementation workspace for Lantern Protocol signal tooling, threat modeling, and protocol experiments.

## Operating principles

- Build for hostile, degraded, and ambiguous information environments.
- Treat provenance, auditability, and reproducibility as first-class requirements.
- Prefer small, reviewable changes over broad undocumented implementation jumps.
- Separate protocol design, implementation, security research, and operational deployment concerns.

## Contribution model

Lantern Protocol is currently in early-stage formation. Public contributions are welcome where repositories explicitly allow them, but protocol-impacting changes should start as proposals or design discussions before code changes.

Security-sensitive findings should follow the responsible disclosure process described in `SECURITY.md` rather than being posted publicly.
